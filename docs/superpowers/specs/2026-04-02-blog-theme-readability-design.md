# 博客主题阅读密度优化设计

日期：2026-04-02

## 背景

当前博客主题的整体气质已经符合预期：克制、轻量、没有多余装饰。需要优化的是阅读体验，而不是重做视觉风格。

现状中的主要问题集中在全站通用样式层：

- 内容版心偏紧，长文阅读时正文、代码块、引用之间的呼吸感不足。
- 标题、日期、正文、列表项之间的垂直节奏不够清晰，页面容易显得“挤在一起”。
- 代码块、引用、表格等阅读组件的边界感偏弱，内容类型切换时层次不够明显。
- 移动端虽然可用，但在小屏下容易重新回到紧凑状态，舒展感不稳定。

## 目标

- 在不大幅改变现有主题风格的前提下，提升全站长文阅读的舒适度。
- 优化正文、列表、代码块、引用、表格、图片等高频阅读组件的层次和节奏。
- 保持现有模板结构基本不变，优先通过全局 CSS 完成优化。
- 让首页、归档页、分类页、标签页和文章页共享一致的阅读基线。

## 非目标

- 不做主题重设计，不更换整体配色方向。
- 不引入目录侧栏、悬浮工具栏、复杂动效或新页面结构。
- 不大规模修改 Jekyll 布局模板，只允许做少量必要的类名或结构补位。
- 不处理内容层面的文案重写或文章信息架构重组。

## 已比较的方案

### 方案 A：轻量舒展

只放宽页宽和行高，改动最小，风险低，但对阅读层次的改善有限。

### 方案 B：舒展耐读

在保留现有主题气质的前提下，同时调整版心、段落节奏、列表留白和代码块边界感。该方案兼顾收益和风险，适合作为本次优化基线。

### 方案 C：更强展示感

进一步增加留白和展示感，更接近一次主题再设计。阅读体验会更从容，但偏离当前站点气质的风险更高。

结论：采用方案 B。

## 设计概览

本次改动以全站通用样式为主，必要时对模板补充少量语义类名。设计分为三层：

1. 阅读骨架：统一版心、间距、正文节奏。
2. 阅读组件：明确代码块、引用、表格、图片、列表的层次。
3. 轻量交互：补强链接状态、焦点状态和小屏响应式规则。

## 设计细节

### 1. 阅读骨架

#### 版心

- 放宽主内容最大宽度，使长文阅读不再局促，但仍保持单列阅读形态。
- 页面左右内边距改为更弹性的规则，桌面端获得更多呼吸感，移动端自动收紧但不压缩正文可读性。
- 维持当前中心单栏布局，不增加侧栏。

#### 正文字号与行高

- 维持当前基准字号附近的视觉大小，不做明显放大。
- 提升正文行高与段落间距，使连续段落更耐读。
- 为 `h1`、`h2`、`h3` 建立更清晰的上下间距规则，确保标题与正文的关系一眼可辨。

#### 垂直节奏

- 统一文章内常见块级元素的上下边距，包括段落、列表、代码块、引用、表格和图片。
- 减少“有的地方过挤、有的地方过空”的不一致。
- 首页和列表页沿用同一节奏体系，但比正文略紧，保证扫描效率。

### 2. 阅读组件

#### 代码块与行内代码

- 保留当前浅底代码风格，不切换到深色或高对比主题。
- 增强代码块内边距、圆角和外部留白，让代码块与正文自然分层。
- 改善长代码块的横向滚动体验，避免贴边和压迫感。
- 行内代码保留轻量强调，不做高饱和高存在感样式。

#### 引用

- 在保持简洁的前提下，略增强引用块的边界感和内边距。
- 确保引用与正文之间的距离稳定，不与段落混成一块。

#### 表格与图片

- 表格保持全宽可读，但提供更稳定的间距和滚动容错。
- 图片继续自适应容器宽度，同时保证与前后正文的留白一致。

#### 列表页条目

- 针对首页、归档页、分类页和标签页的文章列表，增加条目间的纵向节奏。
- 让日期与标题的对齐更稳定，提升快速扫读时的辨识度。
- 长标题允许自然换行，但不让整行显得拥挤。

### 3. 轻量交互与响应式

#### 链接与焦点反馈

- 保留当前链接主色和基础风格。
- 补充更明确的 hover 与键盘 focus 状态，让阅读中断到点击行为的切换更清楚。
- 已访问链接继续与未访问链接保持可区分状态。

#### 小屏规则

- 移动端优先保住正文行高和内容块分层，其次再压缩页边距。
- 列表项、代码块、表格在小屏下优先防止内容拥挤和视觉贴边。
- 不额外引入移动端专属复杂组件。

## 模板与样式边界

优先修改下列位置：

- `assets/style.css`

必要时允许在下列模板中增加少量类名或包裹结构，以便更稳定地控制样式：

- `_layouts/default.html`
- `_layouts/post.html`
- `_includes/post_list.html`

不调整内容渲染逻辑，不改变菜单信息架构。

## 实现原则

- 优先 CSS 改动，最小化 HTML 结构变动。
- 复用现有颜色体系和字体设置，不做主题级视觉跳变。
- 新增样式尽量基于已有元素选择器和少量明确类名，不堆叠脆弱选择器。
- 响应式规则以小屏可读性优先，而不是简单缩小所有间距。

## 风险与应对

### 风险 1：留白增加后首页显得变慢

应对：列表页节奏只做中度放松，确保仍然适合扫描；正文页的舒展程度高于列表页。

### 风险 2：版心放宽后代码块或图片显得松散

应对：代码块和图片独立设置内外边距与最大宽度表现，避免仅靠版心变化被动适配。

### 风险 3：移动端重新变挤

应对：单独设计小屏边距和组件间距规则，而不是完全继承桌面端比例。

## 验收标准

- 首页和文章页在首屏即能感受到更明确的层次和更自然的留白。
- 长文正文连续阅读时，段落、标题、代码块、引用之间的切换更轻松。
- 文章列表的日期与标题更容易扫读，不出现明显拥挤或断裂。
- 移动端阅读不贴边，代码块与表格仍可操作。
- 整站仍然保留现有简洁、克制的主题气质，用户不会感觉这是一个新主题。

## 测试与验证

- 本地构建站点后，检查首页、文章页、归档页、分类页和标签页。
- 分别在桌面宽屏、常见笔记本宽度和手机宽度下验证阅读节奏。
- 至少选择一篇正文较长、包含代码块和图片的文章进行人工走查。
- 检查链接 hover、focus、visited 状态是否仍然清楚且不破坏整体风格。

## 交付范围

本次只交付阅读密度优化所需的样式与少量模板补位，不包含新的内容模块、功能入口或主题切换能力。
