# 首页展示优化计划

## 背景

当前首页延续了 `no-style-please` 的极简结构，整体方向是对的：信息直接、加载轻、维护成本低。

不过从首页的展示效率来看，仍有几个可以继续优化的点：

- 首页承担了“入口页”和“目录页”的双重角色，导航信息略分散。
- 菜单文案存在中英文混用，整体气质不够统一。
- `tag` 列表信息密度较高，容易稀释“最近文章”的注意力。
- `github repo` 等补充链接占据了首页首屏资源，但不是大多数读者的首要动作。

本次优化的前提是不破坏“简洁”这一核心特征，只做轻量整理和层级优化。

## 目标

1. 保持首页结构简洁，不引入复杂模块、摘要卡片或过重视觉装饰。
2. 提高首屏可读性，让读者更快理解站点主题和主要入口。
3. 突出“最新文章”这一核心内容，降低次要信息的干扰。
4. 统一首页文案语言和信息组织方式，提升整体完成度。

## 优化原则

- 少即是多：优先删除或下沉信息，而不是继续往首页添加内容。
- 主次分明：首页优先承载“我是谁、这里写什么、最近有什么”。
- 一屏可扫读：首屏尽量让用户快速完成理解和第一次点击。
- 保持可维护性：优先修改数据结构、模板和少量样式，避免引入复杂逻辑。

## 建议方案

### 1. 收缩首页信息结构

将首页控制在 3 个核心区块内：

- 站点标题 + 一句简短说明
- 最新文章
- 次级入口（分类、关于、订阅）

建议弱化或移除以下内容的首页优先级：

- `posts by tag`
- `github repo`
- `another list` 这类语义较弱的分组标题

其中：

- `tag` 更适合放在独立页面中承接深入浏览。
- `github repo` 更适合放到 `About` 页面或首页底部较弱位置。
- “关于”建议保留，但作为轻量入口，不必单独占据一个松散分组。

### 2. 增加一句站点说明

当前首页仅展示标题，缺少一句帮助读者快速建立预期的描述。

建议启用 `site.description`，文案控制在一行以内，例如：

- 记录工程实践、前端、游戏开发与效率工具
- 关于编程、技术写作与个人知识管理的持续记录

说明文案的作用不是“介绍完整”，而是帮助首次访问者快速判断内容是否相关。

### 3. 统一首页文案语言

当前首页存在中英混用，如：

- `all posts`
- `See archive...`
- `another list`
- `rss`

如果站点正文和读者语境以中文为主，建议首页导航统一成中文，例如：

- 最新文章
- 查看归档
- 分类
- 关于
- 订阅

这样能让首页更稳定，也更符合内容主体语言。

### 4. 强化“最新文章”区块的主入口地位

首页最重要的内容应该始终是最近更新。

建议保留“仅显示最近 5 篇”的策略，同时做两点微调：

- 将“查看归档”作为列表后的自然收尾入口
- 适度提升文章标题和日期的层级对比

不建议在首页为文章增加摘要，因为这会明显提高页面长度，并削弱当前的简洁风格。

### 5. 将高密度导航下沉到次级页面

随着文章数量继续增长，`tag` 和更细的导航维度会越来越多。与其让首页承担所有入口，不如把复杂性放到归档页或独立列表页。

建议策略：

- 首页保留最常用入口
- Archive 页承担完整时间线浏览
- Category 页承担主题浏览
- Tag 页继续保留，但不在首页展开

这样既能保证功能完整，也不会让首页逐步膨胀。

### 6. 只做轻量视觉优化

当前样式基础已经比较克制，后续只建议做非常有限的调整：

- 让首页区块之间的垂直间距更明确
- 让说明文字的颜色略淡于标题
- 让日期弱于标题，增强扫读效率
- 让次级入口区域与文章列表有轻微分隔

不建议加入以下元素：

- Banner
- 大面积配图
- 卡片式文章列表
- 复杂 hover 动效
- 多栏布局首页

这些做法虽然常见，但会明显偏离当前站点的简洁路线。

## 推荐落地版本

建议先实现一个最小可行版本：

### 首页结构

- 标题
- 一句描述
- 最新 5 篇文章
- 分类入口
- 关于 / 订阅

### 首页移除或下沉内容

- 首页展开的 tag 列表
- `github repo` 的高优先级位置
- 语义不强的分组标题

## 实施步骤

1. 调整 `_data/menu.yml`，重组首页区块顺序和文案。
2. 调整 `_config.yml`，启用并填写 `description`。
3. 视需要微调 `_layouts/home.html`，让描述和主内容层级更清晰。
4. 微调 `_sass/no-style-please.scss` 或 `_sass/custom.scss`，只做轻量间距和文字层级优化。
5. 本地预览首页首屏、移动端和归档入口，确认没有破坏现有风格。

## 验收标准

- 首页首屏在几秒内可理解站点主题。
- 用户能很快看到最近文章，并自然进入文章或归档页。
- 首页项目数量比现在更少，信息结构比现在更清晰。
- 整体观感仍然是“简洁、克制、技术博客”，而不是“重新设计了一版首页”。

## 后续可选项

如果这一版效果满意，后续可以继续考虑：

- 为 Archive 页做更好的年份分组展示
- 为 About 页补充站点说明、写作主题和外部链接
- 为首页增加“精选文章”入口，但前提是仍保持极少数量

以上可作为第二阶段优化，不建议在本轮一起做。
